# v0.3.0

## Features

- Tracks Codex, CodeBuddy, and WorkBuddy task sessions independently.
- Prevents one task's completion or sleep event from hiding another active task.
- Prioritizes waiting, error, and working states before terminal states.
- Tails multiple active Codex Desktop and VS Code session logs in one watcher.
- Exposes aggregate owner, session identity, status counts, and deadlines from
  the Hub state API.
- Uses native macOS power notifications to force the light off before normal
  sleep and reset it to idle after wake.

## Behavior

- Completion displays for 3 seconds before yielding to other work.
- Error displays for 10 seconds before transitioning to idle.
- Idle transitions to sleeping after 30 seconds.
- Working and waiting sessions use defensive stale leases of 15 and 30 minutes.
- Lifecycle hooks remain non-blocking, and one delivery worker still serializes
  BLE with USB serial fallback.
- System sleep clears all sessions and masks racing lifecycle events. Wake,
  login, supervisor restart, and Hub restart publish idle without restoring old
  work.
- Abrupt Mac power loss cannot send a last command; an independently powered
  ESP32 needs a firmware watchdog for an absolute stale-light guarantee.

## Install or update

Apple silicon users can install or update with the same public command:

```bash
curl -fsSL https://raw.githubusercontent.com/kubai087/codex-clawd-status-macos/main/install.sh | bash
```
