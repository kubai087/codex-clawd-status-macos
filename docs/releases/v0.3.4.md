# v0.3.4

## Fixes

- Restore the yellow waiting light when Codex Desktop asks the user to
  authorize network, filesystem, or escalated command access.
- Recognize the current `tools.request_permissions` API as well as legacy
  direct and nested `exec_command` events, including the current mixed
  JavaScript-object argument syntax used by orchestrated calls.
- Treat explicit approval, user-input, plugin-install, plan-exit, and MCP
  elicitation requests as the same high-priority waiting state across Codex,
  CodeBuddy, and WorkBuddy.
- Resume the existing thinking state after an approval response without
  changing multi-session arbitration, final completion, or macOS sleep rules.
- Scan nested tool calls outside JavaScript strings and comments, avoiding
  false yellow states when a normal command merely mentions approval APIs or
  `require_escalated`.
- Deactivate a Codex session immediately on `turn_aborted`, preventing a stale
  working animation from masking the final green completion of another task.
