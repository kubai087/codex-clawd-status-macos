# v0.3.3

## Fixes

- Keep the final task's successful completion green instead of automatically
  transitioning through idle to sleeping after a few seconds.
- Preserve multi-session arbitration: a task that completes while another task
  is actionable remains transient and cannot hide ongoing work or a prompt
  waiting for attention.
- Clear the final completion latch on new activity, explicit session end,
  macOS sleep, or Hub restart so stale success cannot reappear later.
- Expose `completion_latched` in the Hub client state for diagnostics.
